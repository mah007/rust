A production-grade async web framework
written from scratch in Rust

oxide-web pairs a high-performance asynchronous HTTP server with an ergonomic, Axum-style framework — a radix-tree router, typed responses, graceful shutdown, and structured tracing, all in safe Rust with #![forbid(unsafe_code)].

This very page is being served by an oxide-web server. 🦀

⚡
Async on Tokio & Hyper

HTTP/1.1 and HTTP/2 via Hyper's automatic connection builder — no hand-rolled parsing, no custom crypto.

🌳
Radix-tree router

Static, :param, and *wildcard routes with static > param > wildcard priority and startup conflict detection.

🧩
Ergonomic handlers

Any async fn returning something that implements IntoResponse — text, HTML, status codes, tuples, and more.

🛑
Graceful shutdown

Ctrl+C/SIGTERM stops accepting connections and drains in-flight requests within a bounded timeout.

🔎
Structured tracing

Every request logs method, path, status, latency, and peer — never leaking secrets, headers, or panics to clients.

🧪
Testable by design

An in-process TestServer starts on an ephemeral port with chainable response assertions.

Up and running in seconds

Register routes on the Application builder, bind an address, wire up graceful shutdown, and run. That's the whole loop.

  • ✅ Plain-text & HTML responses
  • ✅ 404 and 405 (with Allow)
  • ✅ Shared application state
  • ✅ Remote peer address in handlers
use oxide_web::{Application, Html, routing::get};

async fn index() -> Html<&'static str> {
    Html("<h1>Hello from oxide-web!</h1>")
}

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    Application::new()
        .route("/", get(index))
        .bind("127.0.0.1:8080")
        .graceful_shutdown(oxide_web::shutdown::ctrl_c())
        .run()
        .await?;
    Ok(())
}
